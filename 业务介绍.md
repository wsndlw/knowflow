# 业务介绍

集团信息系统部（Corporate Information System）负责字节跳动信息系统建设。通过构建领先的信息系统，助力公司业务和组织的长期高效经营与发展。与此同时，在安全、隐私、合规层面持续治理，为公司的稳健发展保驾护航。字节跳动的业务模式复杂，用户规模大。到现在，集团信息系统部已经覆盖人事、财务、法务、采购、审批、职场等多个领域，隐私安全工作同步开展，AIGC创新孵化也在逐步落地实现。

# 课题介绍

## 课题名称

AI 知识库管理平台

## 课题背景

在企业日常运作中，大量业务知识分散在文档、对话、代码等载体中，难以高效复用。希望通过 AI 技术构建一个知识管理平台，让用户和 Agent 都能参与知识的生产与消费，实现知识的高效流转和智能服务。

## 课题目标

构建一个 AI 知识库管理平台，满足以下核心能力：

- 知识的生产：支持多种方式产生知识（用户手动录入、文档导入、AI 自动提炼等，方案不限）

- 知识的管理：支持知识的组织、分类、检索和基础管理

- 知识的消费：基于知识库提供智能服务能力（可以是生成专家 Agent 进行对话问答，也可以是生产 Skill 供其他 Agent 调用，形式不限）

## 基础要求

### 知识管理

- 提供知识的创建、编辑、删除等基础管理能力

- 支持对知识进行检索（方案不限：关键词搜索、语义搜索、标签筛选等均可）

- 知识条目有基本的状态区分（如可用/不可用）

### 知识消费

- 基于知识库对外提供智能服务（以下任选其一或组合）：

- 方式一：生成可对话的专家 Agent，回答基于知识库内容

- 方式二：生产 Skill/插件，可在其他 Agent 平台上使用

- 方式三：其他合理的知识消费形式

### 基础体验

- 管理界面操作路径清晰

- 知识消费端（对话/调用等）响应流畅

## 加分项

- Agent 或系统可从使用过程中自动提炼新知识，形成生产闭环

- 支持多模态知识（图片、表格等）的录入与理解

- 知识使用热度统计与可视化

- 能够根据知识库一键生成专家 Agent，并实际完成问答流程

## 技术栈（建议）

以下技术栈仅为建议，**不做强制要求**。学员可根据课题特点和团队能力自由选择技术方案，评审时更看重方案的合理性和最终效果。

- **前端框架：**React、Vue、Next\.js、Svelte 等均可

- **后端服务：**Node\.js、Python、Go 等均可

- **开发语言：**TypeScript（推荐）、JavaScript、Python 等均可

- **UI 组件库：**不限（Ant Design、Semi Design、Shadcn、Material UI 等自由选择）

- **AI 能力：**火山引擎、OpenAI、Coze、阿里云百炼等 MaaS 平台均可，也可自行部署开源模型

- **数据库/存储：**不限（SQLite、PostgreSQL、MongoDB、Supabase 等均可）

## 工程规范

- 项目代码存放于 Git 仓库，结构清晰

- 建议集成代码质量工具（如 ESLint、Prettier 等）

- 代码具备基本的可读性和可维护性

- README 包含项目介绍、启动方式、架构说明、分工说明

- 鼓励使用 Trae、Cursor、Codex 等 AI 编程工具提升效率，但需要对架构设计和代码质量有自己的把控
